Vue Components

What are Components?

Components are reusable pieces of UI. Think of them as custom HTML elements.

Small components combine to make bigger features

1. Importing Components

Before using a component, you must import it:

<script setup>
// Import from Components folder
import PrimaryButton from '@/Components/PrimaryButton.vue';
import TextInput from '@/Components/TextInput.vue';
import InputLabel from '@/Components/InputLabel.vue';

// Import from Layouts folder
import AuthenticatedLayout from '@/Layouts/AuthenticatedLayout.vue';

// Import from Inertia
import { Head, Link } from '@inertiajs/vue3';
</script>
@ symbol: The @ is an alias for resources/js/. So @/Components/ means resources/js/Components/.

2. Props - Passing Data to Components

Props are data passed from a parent component to a child component.

Defining Props (in the child)

// NavLink.vue - receives href and active props
<script setup>
const props = defineProps({
    href: {
        type: String,       // Expected data type
        required: true,     // Must be provided
    },
    active: {
        type: Boolean,      // true or false
        // Not required - optional prop
    },
});

// Access props with: props.href, props.active
</script>

Passing Props (from the parent)

<!-- In AuthenticatedLayout.vue -->
<NavLink 
    :href="route('employee.dashboard')"  <!-- Dynamic value -->
    :active="route().current('employee.dashboard')"
>
    Dashboard
</NavLink>

Static vs Dynamic Props

Syntax Type Example
prop="value" Static string type="text"
:prop="value" Dynamic JavaScript :min="today"
:prop="true" Boolean true :active="true"

Real Example: Employee Dashboard

// Employee Dashboard receives these props from Laravel
const props = defineProps({
    balances: Array,    // Array of leave balance objects
    requests: Object,   // Paginated leave requests
    user: Object,       // Current user data
});

// Use them in the template:
<div v-for="balance in balances">
    {{ balance.leave_type }}
</div>

3. Slots - Passing Content to Components

Slots let you pass HTML content into a component.

Default Slot

<!-- PrimaryButton.vue -->
<template>
    <button class="...">
        <slot />  <!-- Content goes here -->
    </button>
</template>

<!-- Usage -->
<PrimaryButton>Submit Request</PrimaryButton>
<!-- "Submit Request" replaces <slot /> -->

Named Slots

Use named slots when a component has multiple insertion points:

<!-- AuthenticatedLayout.vue has named slots -->
<template>
    <!-- Named slot for header -->
    <header v-if="$slots.header">
        <slot name="header" />
    </header>

    <!-- Default slot for main content -->
    <main>
        <slot />
    </main>
</template>

Using Named Slots

<!-- Employee/Dashboard.vue -->
<AuthenticatedLayout>
    <!-- Fill the "header" slot -->
    <template #header>
        <h2>Employee Dashboard</h2>
    </template>

    <!-- This goes in the default slot -->
    <div class="py-12">
        ...main content...
    </div>
</AuthenticatedLayout>
Shorthand: #header is the same as v-slot:header

5. Real Component: NavLink.vue

Let's analyze an actual component from your app line by line:

// resources/js/Components/NavLink.vue

<script setup>
// 1. Import Vue's computed function
import { computed } from 'vue';

// 2. Import Inertia's Link component
import { Link } from '@inertiajs/vue3';

// 3. Define what props this component accepts
const props = defineProps({
    href: {
        type: String,
        required: true,
    },
    active: {
        type: Boolean,
    },
});

// 4. Computed property: CSS classes based on active state
const classes = computed(() =>
    props.active
        ? 'border-b-2 border-indigo-400 text-gray-900 ...'  // Active styles
        : 'border-transparent text-gray-500 hover:text-gray-700 ...'  // Inactive
);
</script>

<template>
    <!-- Use Inertia's Link with dynamic href and classes -->
    <Link :href="href" :class="classes">
        <slot />  <!-- Link text goes here -->
    </Link>
</template>

What This Component Does

  1. Receives href (URL) and active (boolean) as props
  2. Computes CSS classes based on whether link is active
  3. Renders an Inertia Link with those classes
  4. The link text comes through the slot

6. Layouts

Layouts are special components that wrap entire pages.

Your Two Layouts

Layout Used For Contains
GuestLayout Login page, Contact Admin Logo, centered card
AuthenticatedLayout All dashboard pages Navbar, user dropdown, flash messages

How Layouts Work

<!-- Login.vue uses GuestLayout -->
<GuestLayout>
    <Head title="Sign In" />
    <form>...login form...</form>
</GuestLayout>

<!-- Dashboard.vue uses AuthenticatedLayout -->
<AuthenticatedLayout>
    <template #header>
        <h2>Employee Dashboard</h2>
    </template>
    
    <div>...dashboard content...</div>
</AuthenticatedLayout>
